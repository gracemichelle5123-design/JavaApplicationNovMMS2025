# Complete List of All 57 Java Keywords with Descriptions

## Important Note
Out of 57 keywords, 55 are in use and 2 are not in use** (const and goto are reserved but not used).

Also, **true**, **false**, and **null** are literals (not keywords), but they cannot be used as identifiers.

1. abstract
Used to declare a class or method as abstract. An abstract class cannot be instantiated directly, and abstract methods have no implementation. Used for abstraction and inheritance.

2. assert
Used to test assumptions about your program. An assertion enables you to verify that a condition is true at a specific point in the code. If the condition is false, the program terminates with an error.

3. boolean
Data type keyword used to declare a variable that can hold only `true` or `false` values. Default value is `false`.

4. break
Used to break out of a loop or switch statement. It immediately terminates the current loop or switch block and resumes execution at the next statement after the loop/switch.

5. byte
Data type keyword used to declare a variable that can hold an 8-bit signed integer value. Range: -128 to 127.

6. case
Used with the `switch` statement to mark blocks of code. It defines a group of statements to execute if a value matches the case label.

7. catch
Used to catch exceptions generated by a `try` block. Must be used after `try` block. Declares a block of statements to execute if an exception occurs.

8. char
Data type keyword used to declare a variable that can hold a single 16-bit Unicode character. Default value is '\u0000'.

9. class
Used to declare a class. Defines a template/blueprint for creating objects with properties (variables) and behaviours (methods).Encapsulate code.

10. const
**Reserved keyword but NOT CURRENTLY USED** in Java. Use `final` instead to create constants.

11. continue
Used to skip the current iteration of a loop and jump to the next iteration. If followed by a label, it continues execution at the labelled loop.

12. default
Used in two ways:
- In a `switch` statement to specify code to execute if no `case` matches
- In Java 8+, used to declare default methods in interfaces

13. do
Used to declare a do-while loop. The loop body executes at least once before the condition is checked.

14. double
Data type keyword used to declare a variable that can hold a 64-bit floating-point number. Provides double precision.

15. else
Used in conditional statements to specify code that executes when an `if` condition is `false`.

16. enum
Used to declare an enumerated type (a set of fixed constants). Enum constructors are always private by default. Keyword added in JDK 5.0.

17. exports (Java 9+)
Used in modular Java to export a package from a module. Only available in Java 9 and later.

18. extends
Used to indicate that a class inherits from (extends) another class. Also used for interfaces extending other interfaces to add methods.

19. final
Used to make a variable, method, or class immutable/non-modifiable:
- **Final variable:** Cannot change its value after initialization
- **Final method:** Cannot be overridden by subclasses
- **Final class:** Cannot be subclassed

20. finally
Used in a try-catch structure. The `finally` block always executes whether an exception is thrown or not.

21. float
Data type keyword used to declare a variable that can hold a 32-bit floating-point number (single precision).

22. for
Used to declare a for loop. Used to execute a block of code repeatedly when the number of iterations is known.

23. goto
Reserved keyword but NOT CURRENTLY USED** in Java. The language doesn't support goto statements for clean code design.

24. if
Used in conditional statements to test if a condition is `true`. If true, executes the associated block of code.

25. implements
Used in a class declaration to specify that the class implements one or more interfaces. Must define all methods declared in the interface.

26. import
Used at the beginning of a source file to make classes and interfaces from other packages available to the current code without using the full package name.

27. instanceof
Used to test whether an object is an instance of a specified class or implements a specified interface. Returns a boolean value.

28. int
Data type keyword used to declare a variable that can hold a 32-bit signed integer. Range: -2,147,483,648 to 2,147,483,647.

29. interface
Used to declare an interface. An interface is a collection of abstract method definitions and constant values that a class can implement.

30. long
Data type keyword used to declare a variable that can hold a 64-bit signed integer. Used when `int` range is insufficient.

31. module (Java 9+)
Used to declare a module inside a Java application. Only available in Java 9 and later as part of the module system.

32. native
Used in method declarations to indicate that the method is implemented in native code (non-Java language like C/C++) using the Java Native Interface (JNI).

33. new
Used to create a new instance (object) of a class. Allocates memory for the object and calls its constructor.

34. package
Used to declare a Java package that includes the classes in that source file. Organizes related classes into groups.

35. private
Access modifier keyword. Indicates that a variable or method can only be accessed within the same class. Most restrictive access level.

36. protected
Access modifier keyword. Indicates that a variable or method can be accessed within the same package and by subclasses (even in different packages). Cannot be applied to classes.

37. public
Access modifier keyword. Indicates that a variable or method can be accessed from anywhere (most permissive access level). Has the widest scope among all modifiers.

38. requires (Java 9+)
Used to specify the required libraries inside a module. Only available in Java 9 and later for module declarations.

39. return
Used to exit from a method and optionally return a value. The return value must match the method's declared return type.

40. short
Data type keyword used to declare a variable that can hold a 16-bit signed integer. Range: -32,768 to 32,767.

41. static
Used to declare class-level variables or methods that belong to the class itself (not to instances). Static members are shared among all objects of the class.

42. strictfp
Used to restrict floating-point calculations to ensure portability across different platforms. Guarantees consistent floating-point results. Keyword was introduced with Java 1.2.

43. super
Reference keyword used to refer to the immediate parent class object. Used to invoke parent class methods or constructors.

44. Switch
Used to evaluate a variable against multiple values (cases). Executes different blocks of code based on which case matches the variable.

45. synchronized
Used to indicate critical sections or methods in multithreaded code. Ensures that only one thread at a time executes the synchronized code block or method.

46. this
Reference keyword used to refer to the current object in a method or constructor. Used to access instance variables and methods of the current object.

47. throw
Used to explicitly throw an exception during program execution. Can throw any exception class that implements the `Throwable` interface.

48. throws
Used in method declarations to specify which exceptions can be thrown by the method. Declares exceptions that are not handled within the method but passed to the caller.

49. transient
Used in serialization to indicate that a field should not be serialized. When an object is serialized to a stream, transient fields are not included.

50. try
Used to define a block of code that may throw exceptions. Must be followed by `catch` and/or `finally` block. Enables exception handling.

51. void
Used in method declarations to indicate that the method does not return any value.

52. volatile
Used in variable declarations (typically for multithreading) to specify that a variable is modified asynchronously by concurrently running threads. Ensures visibility of changes across threads.

53. while
Used to declare a while loop. Executes a block of statements repeatedly while a condition is `true`. Used when the number of iterations is not fixed.

54. var(Java 10+)
Local variable type inference keyword. Allows declaring local variables without explicitly specifying the type; the compiler infers it from the assigned value. Cannot be used as a type name.

55. record(Java 14+)
Used to declare a special immutable data carrier class that automatically generates constructors, getters, equals(), hashCode(), and toString() methods. Introduced in Java 14.

56. yield(Java 14+)
Used in switch expressions to return a value from a switch branch. Introduced in Java 14 (preview mode). Different from traditional switch statements.
